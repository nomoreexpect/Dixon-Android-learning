## 🔴 第一部分：Activity 生命周期

### 单个 Activity 的完整生命周期

```
     用户打开 App
          ↓
      onCreate()   ← 只调用一次，做初始化
          ↓
      onStart()    ← Activity 可见，但无法交互
          ↓
      onResume()   ← Activity 在前台，可以交互 ★ 常用
          ↓
    [用户使用 App]
          ↓
      onPause()    ← Activity 失去焦点（如弹出对话框）★ 常用
          ↓
      onStop()     ← Activity 完全不可见（被新 Activity 遮盖）
          ↓
   [长时间不用，系统可能在此回收]
          ↓
      onDestroy()  ← Activity 销毁（finish() 或系统回收）
```

### 两个 Activity 跳转时的顺序

```
[启动 SecondActivity]
  MainActivity.onPause()          ← 第一步：A 先暂停
  SecondActivity.onCreate()       ← 第二步：B 开始创建
  SecondActivity.onStart()
  SecondActivity.onResume()       ← B 可交互
  MainActivity.onStop()           ← B 完全可见后，A 才停止

[从 SecondActivity 返回]
  SecondActivity.onPause()
  MainActivity.onRestart()        ← ⚠️ 注意这里是 onRestart，不是 onCreate
  MainActivity.onStart()
  MainActivity.onResume()
  SecondActivity.onStop()
  SecondActivity.onDestroy()      ← finish() 的话才会 Destroy
```

### 关键记忆点

| 方法 | 触发时机 | 常见操作 |
|------|---------|---------|
| `onCreate` | 第一次创建 | 初始化视图、加载数据 |
| `onResume` | 进入可交互状态 | 注册广播、开始动画 |
| `onPause` | 失去焦点 | 注销广播、保存草稿（轻量操作！）|
| `onStop` | 完全不可见 | 释放重量级资源 |
| `onDestroy` | 销毁 | 释放所有资源，防止内存泄漏 |

## 🟡 第二部分：Service 生命周期

### 启动式 Service（startService）

```
startService() / startForegroundService()
       ↓
  onCreate()   ← 第一次创建时调用
       ↓
  onStartCommand()  ← 每次调用 startService 都触发（可多次）
       ↓
  [Service 在后台运行，Activity 死了它还活着]
       ↓
  stopService() 或 stopSelf()
       ↓
  onDestroy()
```

### 绑定式 Service（bindService）

```
bindService()
       ↓
  onCreate() + onBind()  ← 返回 IBinder 给 Activity
       ↓
  [Activity 通过 IBinder 直接调用 Service 方法]
       ↓
  unbindService()（所有绑定者都解绑）
       ↓
  onUnbind() → onDestroy()
```

### 前台 Service 要点

- 必须显示一个通知栏通知（`startForeground(id, notification)`）
- Android 8.0+ 必须用 `startForegroundService()` 启动，且在 **5 秒内** 调用 `startForeground()`
- Android 14+ 需要在 Manifest 声明 `foregroundServiceType`

## 🟢 第三部分：BroadcastReceiver 生命周期

### 两种注册方式对比

| | 静态注册（Manifest）| 动态注册（代码）|
|--|--|--|
| App 未运行时 | ✅ 可以收到 | ❌ 收不到 |
| Android 8.0+ 隐式广播 | ❌ 不允许 | ✅ 允许 |
| 注销 | 系统自动管理 | 必须手动 unregisterReceiver |
| 适用场景 | 开机自启、接收短信等显式广播 | 网络变化、电量变化等 |

### 动态注册规范（本项目做法）

```java
// ✅ onResume 中注册
networkReceiver = new NetworkChangeReceiver();
registerReceiver(networkReceiver, new IntentFilter(ConnectivityManager.CONNECTIVITY_ACTION));

// ✅ onPause 中注销（与注册配对！）
unregisterReceiver(networkReceiver);
```

### onReceive 注意事项

- 运行在**主线程**，超过 **10 秒** 会 ANR
- 不能做网络请求、数据库操作等耗时任务
- 如需异步，使用 `goAsync()` 或启动一个 Service

## 🔵 第四部分：ContentProvider 机制

### URI 格式

```
content://com.example.lifecycledemo.provider/local_data       ← 查全表
content://com.example.lifecycledemo.provider/local_data/3     ← 查 id=3 的记录
         └── authority ──────────────────────┘  └── path ──┘  └── id ┘
```

### 六个必须实现的方法

| 方法 | 对应 SQL |
|------|---------|
| `query()` | SELECT |
| `insert()` | INSERT |
| `update()` | UPDATE |
| `delete()` | DELETE |
| `onCreate()` | 初始化数据库 |
| `getType()` | 返回 MIME 类型 |

### 调用方式（在 Activity 中）

```java
// 获取 ContentResolver，然后通过 URI 操作数据
Cursor cursor = getContentResolver().query(
    LocalDataProvider.CONTENT_URI,  // URI
    null,    // projection（null = 查所有列）
    null,    // selection（WHERE 条件）
    null,    // selectionArgs
    null     // sortOrder
);
```

### ContentProvider 初始化时机

```
App 启动顺序：
  ContentProvider.onCreate()   ← 最早！
  Application.onCreate()
  Activity.onCreate()
```

## 🛠️ 如何运行本项目

1. **用 Android Studio 打开** `AndroidLifecycleDemo/` 目录
2. **音频文件已就绪**：`res/raw/sample_music.wav`（C大调8秒旋律，自动生成）
3. **准备启动图标**：项目使用系统默认图标，也可替换 `res/mipmap/` 下的文件
4. **连接手机或启动模拟器**，点击 Run
5. **打开 Logcat**，过滤标签 `Lifecycle`，观察每个操作触发的生命周期回调

## 📊 四大组件对比速查

| 组件 | 核心职责 | 生命周期关键点 |
|------|---------|--------------|
| Activity | UI 界面与用户交互 | onCreate/onResume/onPause/onDestroy |
| Service | 后台任务（无界面）| onCreate/onStartCommand/onBind/onDestroy |
| BroadcastReceiver | 响应系统/App 广播事件 | onReceive（临时存在，完成即销毁）|
| ContentProvider | 跨组件/跨 App 数据共享 | onCreate（应用启动时初始化）|
